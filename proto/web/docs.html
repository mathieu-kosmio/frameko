FRAMEKO — Documentation utilisateur

Frameko rapproche les référentiels de certification autour d'un socle commun de critères. Il permet de chercher une exigence, de comparer deux labels, et de s'auto-évaluer une fois pour satisfaire plusieurs référentiels à la fois.

1. Le principe en une minute

Neuf référentiels horticoles (Florverde, PlanetProof, Fairtrade, Plante Bleue…) décrivent souvent les mêmes exigences avec des mots différents. Frameko relie chaque critère de chaque référentiel à un critère commun (51 au total), via un degré de rapprochement :

DegréSignification
equivautAcouvre la même exigence à l'identique
plusStrictQuel'exigence est plus précise/exigeante que le critère commun
plusLargeQuel'exigence couvre un périmètre plus large
rapprocheDeproche, sans être équivalent ni strictement comparable

2. Les trois services (console web)

Recherche sémantique

Décrivez une exigence en langage libre (ex. « économie de l'eau d'irrigation »). Frameko renvoie les critères communs les plus proches, puis les exigences correspondantes des référentiels avec leur degré. La recherche est sémantique : elle trouve les formulations proches même sans les mêmes mots. Un filtre permet de se restreindre à un référentiel.

Comparaison de couverture

Choisissez deux référentiels : Frameko affiche le nombre de critères communs partagés, et ceux propres à chacun. Utile pour estimer l'effort de passage d'un label à un autre.

Auto-évaluation de conformité

Choisissez un référentiel cible, puis répondez au niveau des critères communs (conforme, partiel, non conforme, non applicable). Comme les réponses portent sur le socle commun, une réponse vaut pour tous les référentiels rattachés au même critère. Frameko calcule un taux de couverture et liste les écarts.

Bénéfice clé : une entreprise multi-certifiée répond une fois et voit immédiatement ce qui est déjà acquis pour les autres labels.

Confidentialité. Les auto-évaluations sont privées et cloisonnées par organisation : on s'y connecte avec un jeton d'organisation, et l'isolation est garantie au niveau de la base (chaque organisation ne voit que ses propres évaluations). La recherche et la comparaison, elles, portent sur des données publiques de référentiels et restent accessibles sans connexion.

3. Utilisation via un assistant (MCP)

Frameko expose un serveur MCP : un assistant comme Claude peut interroger la base directement. Outils disponibles :

OutilUsage
list_frameworks / get_frameworklister / détailler un référentiel
search_requirementsrecherche sémantique d'exigences
nearest_requirementspour un texte, le critère commun de rattachement + voisins
compare_frameworksrecouvrement entre deux référentiels
propose_mappingrattacher une exigence nouvelle au socle (ingestion assistée)
start_assessment / answer_assessment / get_assessment_resultauto-évaluation

4. Ajouter un nouveau référentiel (ingestion)

Le pipeline se déroule en trois temps, sans écrire en base avant validation :

# 1. Extraire les exigences d'une source (tableur ou PDF)
python scripts/extract_source.py --source source.xlsx --out criteria.csv

# 2. Proposer le rattachement au socle commun (degré décidé par l'agent, ou --llm)
python scripts/ingest_framework.py --csv criteria.csv --slug mon-label --title "Mon Label"
#    → produit proposals.json (à relire / compléter les degrés)

# 3. Appliquer une fois validé
python scripts/apply_ingestion.py --proposals proposals.json

La validation humaine (ou par l'agent) entre les étapes 2 et 3 est obligatoire : l'étape apply refuse toute proposition sans degré ou avec un critère commun inconnu.
